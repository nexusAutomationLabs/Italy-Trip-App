Per D-16 and UI-SPEC:

Structure: Server Component that fetches events, passes to a client MapView component.

```tsx
// src/app/(app)/map/page.tsx
import { createClient } from '@/lib/supabase/server'
import { MapView } from './_components/MapView'

export default async function MapPage() {
  const supabase = await createClient()
  const { data: events } = await supabase
    .from('events')
    .select('id, title, category, event_date, latitude, longitude, address')
    .not('latitude', 'is', null)
    .not('longitude', 'is', null)
    .order('event_date', { ascending: true })

  return (
    <div className="h-[calc(100vh-64px)] lg:h-screen w-full">
      <MapView events={events ?? []} />
    </div>
  )
}
```

1. Create `src/app/(app)/itinerary/_components/AddressAutocomplete.tsx` — client component wrapping Google Places Autocomplete.

Per D-18 and research Pattern 4:

```tsx
'use client'

import { useRef, useEffect } from 'react'
import { useMapsLibrary } from '@vis.gl/react-google-maps'
import { Input } from '@/components/ui/input'

interface AddressAutocompleteProps {
  defaultValue?: string
  onSelect: (address: string, lat: number, lng: number) => void
  onChange?: (value: string) => void
}
```

Implementation:
- Use `useMapsLibrary('places')` to get the places library
- On mount (when places library loads), create a `google.maps.places.Autocomplete` instance on the input ref
  - Fields: `['formatted_address', 'geometry']`
  - Restrict to Italy for better results: `componentRestrictions: { country: 'it' }`
- On `place_changed` event:
  - Get `place.formatted_address`, `place.geometry.location.lat()`, `place.geometry.location.lng()`
  - Call `onSelect(address, lat, lng)`
- Input styled as shadcn Input with `placeholder="Search for a place or address"` (UI-SPEC copy)
- If places library not available (no API key), render a plain Input that just calls `onChange` — degrades gracefully

Note: This uses the legacy `google.maps.places.Autocomplete`. Per research, if the user's Google Cloud account was created after March 2025, this may not be available. In that case, fall back to `PlaceAutocompleteElement` via `@vis.gl/react-google-maps`. The executor should test after implementation and switch if needed.
