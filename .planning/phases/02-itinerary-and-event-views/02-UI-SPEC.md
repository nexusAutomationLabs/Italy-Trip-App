## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline badge padding |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, event row internal gaps |
| md | 16px | `gap-4` / `p-4` | Default element spacing, card internal padding |
| lg | 24px | `gap-6` / `p-6` | Day card padding, section padding |
| xl | 32px | `gap-8` / `p-8` | Layout gaps between day cards |
| 2xl | 48px | `gap-12` / `py-12` | Hero bottom padding, major section breaks |
| 3xl | 64px | `gap-16` / `py-16` | Page-level top/bottom padding |

**Exceptions:**
- Touch targets on event rows: minimum 44px height on mobile (achieved via `min-h-11` + `py-3`)
- Slide-out panel width (desktop): `w-[45%] max-w-xl` — not a spacing token, a layout constraint from D-05
- Mobile bottom sheet drag handle: 4px height, 32px width — matches xs/xl tokens

## Typography

| Role | Font Family | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Body | Montserrat (`font-sans`) | 14px (`text-sm`) | 400 (regular) | 1.5 | Event descriptions, panel body text, attendee count |
| Label | Montserrat (`font-sans`) | 14px (`text-sm`) | 600 (semibold) | 1.4 | Event titles in rows, time displays, location text |
| Heading | Playfair Display (`font-heading`) | 20px (`text-xl`) | 600 (semibold) | 1.2 | Day headers ("Thursday, May 7"), panel event title |
| Display | Playfair Display (`font-heading`) | 28px (`text-3xl`) | 600 (semibold) | 1.1 | Hero section title ("Berwick goes to Tuscany 2026") |

**Rules:**
- Headings always use `font-heading` (Playfair Display). All other text uses `font-sans` (Montserrat).
- Only two weights used: 400 (regular) and 600 (semibold).
- Day format: `format(new Date('${isoDate}T12:00:00'), 'EEEE, MMMM d')` — e.g. "Thursday, May 7". Source: D-02, RESEARCH.md Pattern 3.
- Event times: 12-hour format with am/pm, e.g. "7:00 PM". Use `date-fns` `format(time, 'h:mm a')`.

## Interaction Contracts

### Event Row — click to open panel
- **Trigger:** Click anywhere on the EventRow
- **Response:** Sets `selectedEvent` in ItineraryClient state — opens Sheet (desktop) or Drawer (mobile)
- **Keyboard:** Enter and Space activate; event row is `role="button"` or a `<button>` element
- **Focus:** Focus moves to panel title on open; returns to triggering row on close
- **Visual state:** Hover — subtle background tint (`hover:bg-muted/40`); no persistent selected state on row

### Event Detail Panel — desktop Sheet
- **Breakpoint:** `min-width: 768px` (Tailwind `md:`)
- **Width:** `w-[45%] max-w-xl`
- **Side:** `right`
- **Animation:** Slide in from right, 200ms ease-out (shadcn default `tw-animate-css`)
- **Close triggers:** Click overlay backdrop, press Escape, click X button in panel header
- **Scroll:** Panel content scrolls independently if event description is long

### Event Detail Panel — mobile Drawer (bottom sheet)
- **Breakpoint:** below `768px`
- **Direction:** bottom
- **Drag handle:** visible at top center, 4px × 32px, color `--muted-foreground`
- **Snap points:** none — full screen only (consistent with "full-screen bottom sheet" from D-05)
- **Close triggers:** Swipe down past threshold, tap backdrop, press Escape

### Google Maps location link (D-07)
- **Display:** `<MapPin size={16} />` icon inline-flex with location name text
- **Color:** `text-primary` (terracotta) with `hover:underline`
- **Target:** `target="_blank" rel="noopener noreferrer"`
- **Conditional:** Only rendered when `location_url` is non-null — no empty link placeholder

## Layout Contracts

### Itinerary Page Structure
```
<page>
  <HeroSection>                         ← existing Tuscany golden hour background image
    title: "Berwick goes to Tuscany 2026"
    subtitle: trip dates or countdown (from Phase 1)
  </HeroSection>
  <ItineraryClient>                     ← 'use client' wrapper
    <div className="max-w-3xl mx-auto px-4 py-8 space-y-8">
      {10 × <DayCard>}                  ← one per day, May 7–16
    </div>
    <EventDetailPanel />                ← Sheet or Drawer, portaled
  </ItineraryClient>
</page>
```

### DayCard Structure
```
<Card className="[border-l-4 border-primary if arrival/departure]">
  <CardHeader>
    <h2 className="font-heading text-xl font-semibold">Thursday, May 7</h2>
    [arrival/departure badge if applicable — text-sm, muted]
  </CardHeader>
  <CardContent>
    {events.map(<EventRow />)}
    [empty state if no events for that day]
  </CardContent>
</Card>
```

### EventRow Structure
```
<button className="w-full flex items-center gap-3 py-3 px-0 hover:bg-muted/40 rounded-md text-left">
  <span className="text-sm font-sans text-muted-foreground w-16 shrink-0">
    7:00 PM
  </span>
  <span className="flex-1 text-sm font-semibold font-sans text-foreground truncate">
    Event Title
  </span>
  <Badge className="{CATEGORY_STYLES[category]} text-xs shrink-0">
    Dinner
  </Badge>
  <span className="text-xs text-muted-foreground shrink-0">
    3 attending
  </span>
</button>
```

### EventDetailPanel Content Order
1. Category badge (top, left-aligned)
2. Event title (font-heading, text-xl, font-semibold)
3. Date + time (text-sm, muted-foreground)
4. Description (text-sm, body, line-height 1.5)
5. Location link (MapPin icon + text, text-primary, only if location_url exists)
6. Attendee count summary ("3 attending" — text-sm, muted-foreground)

## Accessibility Contract

- All event rows are `<button>` elements (not `<div onClick>`) — keyboard accessible by default
- Sheet and Drawer handle focus trapping and ARIA roles via shadcn/base-ui primitives — do not re-implement
- Category badges: color is not the only differentiator — text label always present alongside color
- Location links: include `aria-label="Open {location_name} in Google Maps"` for screen reader context
- Day headers use semantic `<h2>` within card header
- Attendee count includes screen-reader-friendly text: "3 people attending" (visible text can abbreviate)
